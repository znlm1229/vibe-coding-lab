# Stage 1 ｜ Brainstorm 头脑风暴

> 详细规范见 [workflow-spec](../../../../workflow-spec/specification.md#阶段-1--brainstorm-头脑风暴)
>
> **要点**：发散 3–5 个方向；每个配一句核心思路 + 一句主要权衡；**不评判出胜者**（由用户决定）。

## 任务

做一个属于用户个人的个人网站。

用户原话："做一个属于我个人的个人网站"。范围、定位、技术栈未指定，故本阶段从「这是个什么样的个人网站」入手发散，让选项空间摊开，避免直接锚定到某一种形态。

## 方向

### A — 极简一页静态名片

- **核心思路**：单页 HTML/CSS，about / skills / contact 三段栈式排版；纯静态，托管在 GitHub Pages / Cloudflare Pages；无博客、无后端、无 JS 框架。
- **主要权衡**：上线最快 / 维护成本最低；表达力天花板低，几乎不可扩展成持续内容平台。

### B — 博客驱动（写作中心）

- **核心思路**：博客作为站点核心，about 是次要。技术栈选 Hugo / Astro / Next.js + MDX 等静态生成器；按主题分类的文章列表 + 标签 + RSS。
- **主要权衡**：长期能沉淀价值与个人品牌；成败完全取决于"是否真的持续写"，没内容时比极简名片更显空架子。

### C — 作品集驱动（项目中心）

- **核心思路**：项目展示作为站点核心，工作 / 业余项目以卡片网格陈列，每个项目独立详情页（截图、技术栈、链接、复盘）；about 是入口而非主菜。
- **主要权衡**：求职 / 接私单 / 出圈最直接的形态；但需要"真的有项目可展示"，每个还要拍图 / 录屏 / 写描述，前期一次性投入大。

### D — 混合综合站（about + 博客 + 作品集 + 联系）

- **核心思路**：传统 personal website 形态，什么都有一点：首页 hero + 关于 + 精选项目 + 最近博客 + 联系入口；技术栈选 Next.js / Astro 这类支持混合内容类型的多页静态生成器。
- **主要权衡**：表达维度最完整、可塑性高；维护工作量最大，每一块都可能弱（博客没空写、作品集没空补、联系信息长期不更新都很常见）。

### E — SaaS 寄生（Notion / Substack / GitHub Profile / Linktree 等托管服务）

- **核心思路**：不自建，直接用 Notion / Substack / Medium / GitHub Profile README / Linktree / Carrd 等托管服务做个人页，通过自定义域名指过去。
- **主要权衡**：零代码零运维、可立即上线；定制度低、品牌感弱、被平台绑死；并且几乎不沾代码，对 vibe-coding-lab 的"练手九步流程"目的贡献有限。

## 给用户的建议下一步

按 Stage 1 规则**不评判胜者**。请你：

1. **告诉我哪个（或哪些组合）方向值得推进到 Stage 2** —— 也可以排除你明显不想要的
2. 我会用 `grill-me` skill 围绕你选中的方向拷问：定位 / 受众 / 必要内容 / 技术栈 / 域名 / 托管 / 维护频率 / 设计风格 / 不做什么 等决策树分支
3. 拷问结果汇成 Stage 2 artifact，喂给 Stage 4 SPEC

> 如果想看几个方向的混合（如 B+C 博客 + 作品集，或 A+E 极简 + 托管），也可以指定。
