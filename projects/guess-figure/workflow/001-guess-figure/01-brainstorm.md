# Stage 1 ｜ Brainstorm 头脑风暴

> 详细规范见 [workflow-spec](../../../../workflow-spec/specification.md#阶段-1--brainstorm-头脑风暴)
>
> **要点**：发散 3–5 个方向；每个配一句核心思路 + 一句主要权衡；**不评判出胜者**（由用户决定）。
>
> 本阶段由 `brainstorming` skill 驱动。流程：探索意图（2 轮澄清问答）→ 直接发散方向。**不进入 design / writing-plans**（那是 Stage 4/5 的事）。

## 任务

> **用户原话**："做一个程序：猜历史人物，根据几条提示来猜历史人物"

做一个 **猜历史人物** 的程序：AI 给出若干条线索，玩家根据线索猜出目标人物是谁。

### 已收集的约束（来自本阶段 2 轮澄清）

| 维度 | 决定 | 含义 |
|---|---|---|
| 受众 / 场景 | **公开上线给陌生人玩** | 形态需 Web + 公网部署；要考虑反作弊、频率限制、调用成本 |
| 内容机制 | **混合：题库为主 + LLM/爬虫种子化生产 + 后续 LLM 增量** | 项目起步无题库，需先做"种子化"工具 → 玩家运行时主要读题库（运行时 LLM 调用应尽量轻）→ 长期 LLM 持续往题库增量加新人物 |

> 注：上述约束在 Stage 2 Grill Me 中应被压力测试（成本上限？反作弊到什么程度？题库初始规模目标？），不是不可变契约。

## 方向

> 5 个方向围绕「玩家与 AI 的交互方式」和「内容生产/消费节奏」差异化；每个方向都默认"题库为主 + LLM 生产"的内容机制（除非另注）。

### A — Akinator 反向问答

- **核心思路**：AI 心里默想一个人物，玩家通过自由提问（"是男的吗"、"活在唐朝吗"、"是诗人吗"）逼近答案，AI 用 yes / no / 不告诉你回答，N 次提问内必须猜出。
- **主要权衡**：智力游戏感最强、玩家最主动、沉浸感最好；但**每次玩家自由提问都需要 LLM 实时判断**，运行时 LLM 成本随用户量线性增长，跟"运行时尽量轻"约束直接冲突 —— 除非把题库人物的"问答 yes/no 矩阵"预先离线生产好。

### B — 渐进线索 + 自由输入

- **核心思路**：每局 AI 按难度顺序逐条给出 1–5 条线索（最难的最先给），玩家随时输入答案，越早答中分越高；玩家输入名字时用 LLM 做模糊匹配（"诸葛亮" = "孔明" = "诸葛孔明" = "卧龙"）。
- **主要权衡**：体验最接近现实猜谜节目（电视《一站到底》既视感）；运行时 LLM 只在玩家提交答案时调用一次（模糊匹配），成本可估算；UX 简单；但**对自由输入的容错与异称处理要求高**（错别字、繁简、英文名、别号、谥号都要识别）。

### C — 提示 + 四选一速答

- **核心思路**：每条线索后展示 4 个候选名字（1 正确 + 3 题库内难度相近的干扰项），用时短 / 用提示少得分高，连胜累计 + 排行榜。
- **主要权衡**：体验最像手游 / 上手最快 / 适合移动端病毒传播；运行时**几乎不需要 LLM**（题库需要预存"正确答案 + 干扰项"，干扰项可由 LLM 离线生产时生成）；但"选择题"沉浸感比自由输入弱、容易让人猜运气、稍显幼态。

### D — 每日一题 Wordle 变体

- **核心思路**：全球**同题、每日 1 题、24 小时新换**，玩家分享分数（"今天我用了 3 条线索就猜到"），社交传播是核心留存机制。
- **主要权衡**：留存机制最好（养成每日打开习惯，Wordle 已验证）/ UX 极简 / 病毒传播效应强；但**每天稳定产 1 道高质量题对内容 pipeline 是硬约束**，LLM 出题质量稍微波动就会立刻流失用户；题库初期规模小是致命问题。

### E — 主题包闯关

- **核心思路**：题库切成"主题包"（中国古代帝王 / 西方哲学家 / 二战将领 / 唐宋诗人 / ...），玩家选包闯关，每包 N 题过半算赢。
- **主要权衡**：内容有结构 / 内容生产可批量化（按主题 prompt LLM 一次产一整包，效率高）/ 玩家可按兴趣选；缺点是"闯关包"结构稀释了单局猜谜的紧张感，更像"考试"而非"游戏"，且对新用户首屏要做的选择多了一层（先选包再开始）。

## 给用户的建议下一步

按 workflow-spec Stage 1 原则，**本阶段不评胜者**。但作为下一阶段 (Stage 2 Grill Me) 的输入建议，可考虑以下几种走法：

- **如果想要单一最贴合约束的方向去深拷问** → **方向 B（渐进线索 + 自由输入）** 最贴合「公开上线 + 运行时轻 + 单局有沉浸感」的组合，是 grill-me 单一目标的好候选。
- **如果想要对比拷问两个差异化方向** → 推荐 **B vs C**（自由输入 vs 选择题），这两个在「运行时成本」「上手门槛」「玩家沉浸感」三个轴上权衡截然不同，对比拷问能逼出最关键的产品取舍。
- **如果对社交留存机制更感兴趣** → **方向 D（每日一题）** 单拉出来拷问，重点压力测试"内容 pipeline 能否每日稳定产高质量题"这一关。
- **如果犹豫不决** → 可以请我把"方向 B + 方向 D 元素融合"作为方向 F 加进来再讨论（B 是日常玩法、D 是每日一题作为特殊模式），但**这本身就是 Stage 2 该输出的决策**，不必现在硬加。

**等待用户决定**：拷问哪一个 / 哪几个方向？或要求补 / 改 / 合并方向？
